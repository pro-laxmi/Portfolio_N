## 📂 Folder Structure & Use Cases
Here is a breakdown of what each folder does in your Hugo project:

- **`.github/`**: Contains GitHub Actions workflows. The file inside (`workflows/hugo.yaml`) is responsible for automatically building and deploying your site to GitHub Pages whenever you push code.
- **`content/`**: **Necessary.** This is where your actual portfolio data lives. You write markdown files here for your projects, blog posts, etc.
- **`layouts/`**: **Necessary.** Contains HTML templates that override the default theme's layouts. 
- **`public/`**: **Safe to delete locally.** This folder contains the final built HTML/CSS/JS files generated by Hugo. It gets recreated every time you run the `hugo` command or when GitHub Actions runs. You don't need to commit this to your repo.
- **`static/`**: **Necessary.** Contains static assets like your images (e.g., `me.jpeg`), favicons, and custom CSS/JS. These are copied exactly as-is to the `public` folder during the build.
- **`themes/`**: **Necessary.** Contains the code for the `hugo-profile` theme you are using.
- **`hugo.yaml`**: **Necessary.** The main configuration file for your site.
- **`vercel.json`**: **Safe to delete.** Since you no longer want to use Vercel, this file is unnecessary.

## ❌ Why Images Aren't Loading on GitHub Pages
**Diagnosis:** GitHub Pages hosts your site on a subpath (e.g., `https://pro-laxmi.github.io/Portfolio_N/`). However, in your `hugo.yaml` file, your image paths start with a forward slash `/` (e.g., `/images/me.jpeg`). 
Because of this leading slash, the browser looks for the image at the root of the domain (`https://pro-laxmi.github.io/images/me.jpeg`) instead of within your repository's subpath, causing a 404 Not Found error.

**How to Fix It:**
We can fix this by enabling relative URLs in Hugo. By adding `relativeURLs: true` to your `hugo.yaml`, Hugo will automatically adjust all links to be relative to the current page, ensuring they work perfectly regardless of whether the site is hosted on a root domain or a subpath like GitHub Pages.
